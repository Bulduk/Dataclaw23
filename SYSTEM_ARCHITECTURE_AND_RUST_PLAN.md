# NEXUS (POULS) / DATACLAW - SYSTEM ARCHITECTURE & RUST TRANSITION REPORT

## 1. MİMARİ BAKIŞ AÇISI VE BİNANCE STANDARTLARI (PROGRESSIVE DISCLOSURE)

Platform, kullanıcı deneyimini optimize etmek ve profesyonel borsa standartlarına (Binance, Bybit) ulaşmak için **Kademeli Açıklama (Progressive Disclosure)** prensibine dayanır. Karmaşık algoritmik işlemler, ajan state detayları ve risk yönetim panelleri, yalnızca kullanıcı talep ettiğinde görünür olacak şekilde "Bottom Sheet / Drawer" yapılarına veya modal panellere taşınır.

Web ekosistemimizde, bu native kaydırma ve haptic hissiyatını simüle etmek için `Vaul` tarzı yumuşak geçişli Drawer bileşenleri entegre edilir. Mobil (React Native / Expo) ilerlemelerinde ise doğrudan `BottomSheet` kullanılır.

### Vaul / Bottom Sheet Stratejisi
* **Haptic Katmanı:** Web API `navigator.vibrate` ile desteklenen cihazlarda menü açılışlarına hafif titreşim eklenir.
* **Gizli Karmaşıklık:** Ana trade ekranlarında sadece Al / Sat (Long / Short) butonları ve Engine durumu bulunur. Detaylar (Ajan konfigürasyonları, ince ayarlar) Drawer'lar içinde sunulur.
* **Micro-Interactions:** Yeni sinyal veya ajan mesajları geldiğinde ufak 'Pulse' efektli noktalar kullanılır.

## 2. RUST MİGRASYON VE GÜÇLENDİRME PLANI

Sistemin kalbini oluşturan işlem ve risk motorlarının gücünü maksimize etmek için kritik TypeScript bileşenleri **Rust** ile yeniden yazılıp WASM olarak entegre edilecektir.

### Faz 1: `indicators.ts` -> Rust WASM Crate
* **Amaç:** Matematiksel hesaplamaları (EMA, SMA, RSI, MACD, Bollinger, vb.) hızlandırmak.
* **Uygulama:** Veriler Float64Array olarak WASM belleğine aktarılır, SIMD komutlarıyla paralel işlenir ve sonuçlar `O(1)` bellek kopyalaması ile JS tarafına alınır.

### Faz 2: `policyGuard.ts` -> Rust Native / WASM
* **Amaç:** Saniyenin altında 11 aşamalı güvenlik ve risk kontrolü. (Kill switch, draw-down kontrolü, leverage validasyonu)
* **Uygulama:** Her intent için PolicyGuard çalıştırılır. WASM ile memory-safe ve deterministic sonuçlar garantilenir.

### Faz 3: WebSocket Data Multiplexing -> Tokio + Axum (Backend)
* **Amaç:** Borsa WebSocket bağlantılarından gelen devasa veri akışını (L2 Orderbook, Funding Rates) UI'ı dondurmadan işlemek.
* **Uygulama:** Rust tarafında bir WebSocket multiplexer yazılır. React komponentleri yalnızca render edilmesi gereken sıkıştırılmış frame'leri (delta updates) alır.
